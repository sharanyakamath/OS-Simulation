Introduction

A socket is one of the most fundamental technologies of computer network programming.

A socket represents a single connection between exactly two pieces of software (a so-called point-to-point connection). More than two pieces of software can communicate with client/server or distributed systems by using multiple sockets.

Socket programming is a way of connecting two nodes on a network to communicate with each other. One socket(node) listens on a particular port at an IP, while other socket reaches out to the other to form a connection.The one application that initiates communication is termed the "client" and the other application the "server" Server forms the listener socket while client reaches out to the server.Socket-based software usually runs on two separate computers on the network, but sockets can also be used to communicate locally (inter-process) on a single computer. Sockets are bidirectional, meaning that either side of the connection is capable of both sending and receiving data.

Types of Protocols

  1. Transmission Control Protocol
  2. User Datagram Protocol

Key Features of TCP

  1. Connection oriented protocol
  2. Ordered data transfer: the destination host rearranges according to sequence number-flow control
  3. Error-free data transfer-reliable service
  4. Based on acknowledgments
  5. Generally used when data is to be transferred to a central server in a reliable manner

Key Features of UDP

  1. Connection-less protocol
  2. Unreliable service
  3. No acknowledgements
  4. Used when socket is opened in datagram mode
  5. Message is encapsulated in a datagram
  6. Generally used when client requires a certain service from the server
